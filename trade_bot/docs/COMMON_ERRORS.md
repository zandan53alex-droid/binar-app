# ЧАСТЫЕ ОШИБКИ И ИХ РЕШЕНИЕ (COMMON ERRORS)

Если что-то пошло не так при установке или работе бота, проверьте этот список типичных проблем.

## 1. Белый экран в Telegram Mini App

**Симптом:** При открытии Web App в Telegram появляется только белый экран, ничего не загружается.
**Причина 1:** Не установлен HTTPS сертификат. Telegram требует строго HTTPS.
**Причина 2:** Неверно указан URL в настройках `@BotFather` (например, указан `http://` вместо `https://`).
**Причина 3:** Nginx не запущен или упал с ошибкой.
**Решение:**
1. Проверьте статус Nginx: `systemctl status nginx`
2. Если Nginx не стартует из-за отсутствия файлов сертификата, выполните команду получения сертификата из инструкции `FULL_INSTALL_GUIDE.md`.
3. Убедитесь, что в `@BotFather` веб-адрес начинается с `https://`.

## 2. Nginx не запускается

**Симптом:** `nginx -t` пишет "test failed", или команда `systemctl restart nginx` выдает ошибку.
**Причина 1:** Отсутствуют SSL сертификаты по указанному пути.
**Причина 2:** В конфигурации допущена синтаксическая ошибка (забыта `;` в конце строки).
**Причина 3:** Порт 80 или 443 занят другой программой (например, Apache).
**Решение:**
1. Выполните `nginx -t`, команда покажет в какой именно строке ошибка.
2. Проверьте пути к сертификатам: `ls /etc/letsencrypt/live/ВАШ_ДОМЕН/`. Папка должна существовать.
3. Убедитесь, что порт не занят: `lsof -i :80` или `lsof -i :443`.

## 3. Новости не загружаются ("События временно недоступны")

**Симптом:** Во вкладке новостей надпись "События временно недоступны" или "News temporarily unavailable".
**Причина 1:** Сервер недоступен из интернета. (Забыли включить `postback_service`).
**Причина 2:** Ошибки сети на самом VPS — сервер не может достучаться до `Forex Factory`. (Хотя в системе встроен резервный генератор для таких случаев).
**Решение:**
1. Проверьте статус бэкенда: `systemctl status postback_service`
2. Если он остановлен, запустите его `systemctl start postback_service`.
3. Посмотрите логи: `journalctl -u postback_service -f`, чтобы увидеть причину ошибки. В норме там должно быть `INFO: Forex Factory calendar: X events loaded`.

## 4. Котировки графиков не появляются (график стоит на месте)

**Симптом:** При открытии графика свечи не рисуются, цена не меняется.
**Причина 1:** Не запущен сервер `quotes_service`.
**Причина 2:** `Playwright` (браузер) не может запуститься, потому что не установлены системные зависимости браузера.
**Причина 3:** WebSocket соединение не проксируется через Nginx (отсутствует location `/ws`).
**Причина 4:** Pocket Option временно заблокировал IP вашего сервера (Cloudflare капча).
**Решение:**
1. Проверьте WebSocket: `systemctl status quotes_service`.
2. Посмотрите логи (самое важное!): `journalctl -u quotes_service -f`. Если вы видите `PlaywrightError: Executable doesn't exist`, значит нужно выполнить команду `playwright install`.
3. Если вы видите `browser.new_context: Browser closed` или `Timeout`, возможно Cloudflare блокирует доступ. Сервер автоматически попробует перезайти через несколько секунд.
4. Убедитесь, что зависимости установлены: `npx playwright install-deps` или `playwright install-deps`.

## 5. Бот не отвечает в Telegram

**Симптом:** Пишете `/start`, бот молчит.
**Причина 1:** `bot_service` не запущен.
**Причина 2:** Слишком старый или неверный `TOKEN` бота в файле `.env`.
**Причина 3:** Ошибка в коде Python.
**Решение:**
1. Проверьте статус: `systemctl status bot_service`.
2. Посмотрите логи: `journalctl -u bot_service -n 50`.
3. Уточните `TOKEN` у `@BotFather` и исправьте в `/root/bot/.env`. Не забудьте после этого перезапустить `systemctl restart bot_service`.

## 6. CORS ошибки

**Симптом:** Приложение открывается, график идет, а API запросы (например, новости или /debug) не работают. В консоли браузера (DevTools) красная ошибка `Cross-Origin Request Blocked`.
**Причина:** Вы обращаетесь к API по другому домену/IP (например, открыли по IP, а приложение запрашивает домен).
**Решение:**
Сейчас в коде фронтенда API настроено как `window.location.origin + '/api/...'`, поэтому проблем с CORS не будет, пока вы не попытаетесь обращаться к API с внешнего ресурса.
Если нужно обращаться снаружи, настройте заголовки `add_header 'Access-Control-Allow-Origin' '*';` в `location /api/` в конфиге Nginx.
